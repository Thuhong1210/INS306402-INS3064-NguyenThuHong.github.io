SECTION 1: CORE THEORY (KEY CONCEPTS)

1.1 What is PHP?

PHP stands for PHP: Hypertext Preprocessor (a recursive acronym). It is a server-side scripting language designed specifically for web development. Unlike HTML and CSS which run in the browser, PHP executes on the web server before sending content to the user's browser.

Key Point: PHP is server-side, meaning users cannot see the PHP code—only the HTML output it generates. This makes it ideal for processing data, validating forms, and accessing databases securely.

1.3 PHP Basic Syntax Requirements

PHP code must follow specific formatting rules:

Opening Tag: <?php marks the start of PHP code
Closing Tag: ?> marks the end of PHP code
File Extension: Files must be saved as .php (not .html)
Statements: Each statement must end with a semicolon ;
Comments: Use // for single-line or /* */ for multi-line comments

1.4 Output Statements: echo vs print

Feature echo print
Parameters Accepts multiple parameters Accepts only one parameter
Return Value No return value Returns 1
Speed Faster Slightly slower
Usage More commonly used Less frequently used

Recommendation: Use echo in most cases for optimal performance and flexibility.

1.5 How PHP Works: Request-Response Cycle

When a user requests a PHP file, the following process occurs:

  1. Browser Request: User enters URL or clicks link (e.g., http://localhost/index.php)
  2. Server Receives: Web server (Apache) receives the HTTP request
  3. PHP Processing: Server identifies .php file and passes it to PHP processor
  4. Code Execution: PHP processes code, may query database, performs calculations
  5. HTML Generation: PHP generates HTML output
  6. Response Sent: Server sends HTML response back to browser
  7. Browser Display: Browser renders HTML and displays webpage to user
Critical Concept: The user NEVER sees PHP code—only the HTML output. This is the fundamental difference between server-side (PHP) and client-side (HTML/CSS/JavaScript) processing.

1.6 Embedding PHP in HTML

PHP can be embedded directly within HTML files. The PHP code executes on the server, and only the HTML output is sent to the browser.

<!DOCTYPE html> <html> <head> <title>PHP Example</title> </head> <body> <h1>Welcome</h1> <?php // Method 1: Simple echo echo "Hello World!"; // Method 2: With HTML tags echo "<p>Today is: " . date("Y-m-d") . "</p>"; // Method 3: Shorthand (if enabled) ?> <p>Current time: <?= date("H:i:s") ?></p> </body> </html>

1.7 Comments in PHP

Comments are non-executable text that explain code. They are ignored by the PHP processor:

Single-line: // comment or # comment
Multi-line: /* comment */
Documentation: /** documentation */ for functions and classes

1.8 Best Practices and Common Errors

Best Practice Common Error
Always use <?php opening tag Using short tag <? (may not be enabled)
End statements with semicolon ; Missing semicolon causes parse error
Save files with .php extension Saving as .html prevents PHP execution
Write meaningful comments for complex code No documentation makes code difficult to maintain
Use proper quote matching Mismatched quotes cause syntax errors
